# 言枢 AI (Yanshuai-AI) —— UWP 智能助理与角色扮演系统

言枢 AI (Lumina Yanshuai) 是一款面向 Windows 10/11 的 UWP 原生 AI 客户端与助理应用。本项目包含两个核心子应用：
1. **Lumina.Roleplay (言枢角色扮演)**：专为沉浸式角色扮演设计的客户端，支持角色卡导入导出、基于 BGE-Micro 嵌入模型的本地/云端 RAG 记忆池管理、好感度系统等。
2. **Lumina.Agent (言枢智能代理)**：面向工程效率和多任务的项目助理客户端，支持多文件附件、基于 RAG 的知识库检索、MCP 服务端控制台等。

## 项目架构与技术栈
- **核心框架**：Windows Universal Platform (UWP)
- **底层核心**：Lumina.Core (.NET Standard 类库，共享消息定义及底层数据结构)
- **嵌入计算**：OnDeviceAI / OnDeviceAI2 (底层采用 ONNX Runtime 调用本地 BGE-Micro 模型计算语义向量)
- **语言标准**：C# 7.3 (.NET Native)
- **数据结构**：基于 JSON 进行本地落盘，外置 ImageStore 高效缓存图片，避免大内存占用。

## 最新版本重大改进与 Bug 修复 (2026-07-15)

针对系统稳定性和功能审计报告，我们对两款应用进行了深度重构与修复：

### 1. 备份与恢复系统的事务性重构 (P0-1 ~ P0-4)
- **双重确认**：在覆盖导入 ZIP 备份时，增加弹窗双重确认，防止用户误触清空当前数据。
- **事务性加载**：重写了 ZIP 备份的反序列化与导入流程。采用“先读流反序列化至临时变量，验证全部通过后才更新 DataManager 提交更改”的方案，避免了因损坏备份文件导致的部分配置或数据丢失问题。
- **配置与实体补全**：补全了对 `UserProfiles`、`Projects`、评估结果等关键实体的备份与恢复。

### 2. 消息交互与记忆池逻辑修复 (P0-5, P0-9, P0-11, P0-12, P1-10)
- **用户编辑保护**：修复了在 Agent 中编辑用户消息时，直接生成裸消息导致丢失原消息中的 `ImageRefs` 附件引用的缺陷。
- **防止首句叠加**：去除了在对话重建气泡时可能引起的开场白重复插入的逻辑问题。
- **角色卡删除保护与级联清理**：在 `CharacterCardsPage` 中加入了激活角色卡保护（禁止删除当前聊天的激活角色）。并在删除角色卡时同步调用 `DialoguePoolManager.RemovePool` 清理其记忆池，级联删除所有关联对话，解决数据库膨胀的隐患。
- **记忆池裁剪新近度加权**：针对满载时新记忆无法保存的问题，在 Importance 剪枝算法中引入了时间/新旧加权策略（24小时内 +0.5，7天内 +0.25 分数加成），保护新总结的记忆不被立即裁剪。

### 3. API 解析与交互增强 (P0-7, P1-1, P1-9, P1-16, P1-21, P1-22)
- **Claude 流式 SSE 解析器**：重构了流式解析逻辑，全面适配 Claude `content_block_delta` SSE 协议，解决了 Claude 流式调用时气泡空白的问题，并提升了对无空格 `data:` 规范的兼容性。
- **好感度区间与 RAG 提取修正**：在 `ExtractSection` 提取中补全了好感度头，修复了好感度数字被混入“已确认事实”的严重缺陷，并对好感度变化值增加了 `[0, 100]` 的钳制区间。
- **世界书导入属性补全**：世界书导入增加了对禁用状态（`Disable`）和排序优先级（`Order`）的完整字段映射，并支持其常驻注入顺序的控制。
- **项目与 OOBE 加载同步**：修复了打开项目时不刷新知识库文件列表的 UI 缺陷；修复了 OOBE 设置用户人设和名字当次不生效的列表同步缺陷。

### 4. 健壮性与异常捕获防护 (P0-7, P2-1)
- **发送死锁防护**：将 `SubmitButton` 内的整个发送链路（含图片大图保存 `SaveBase64Async` 等文件IO操作）全部包装于 `try-catch` 中，防止因IO读盘异常引发无征兆闪退或 `_isSending` 死锁。
- **全局异常捕获与友好奔溃界面**：为两个应用配置了全局 `UnhandledException` 和 `UnobservedTaskException` 捕获器，并实现 `CrashLog` 日志落盘（`uwp_crash.log`）。当应用启动失败（如 `OnLaunched` 异常）时，用友好提示面板代替系统默认白屏，显示错误细节。
